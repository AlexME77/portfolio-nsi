Projet collection de films - Mai--Emery Alexandre et Mickaël

Présentation générale

Nous avons décider de réaliser un gestionnaire de films. Il est possible d'importer une bibliothèque de films déja existants ou de partir d'une nouvelle afin de pouvoir ajouter nos films rapidement. Cela peut être utile pour pouvoir faire une liste de films à voir puis de les marqué comme "vu" après les avoir regarder et ensuite utilisé ce gestionnaire afin de répertorier tout les films que l'on a déjà vu.

Organisation du travail

La séparation du travail s'est faite naturellement au fur et à mesure de l'avancement de ce projet :

  • Alexandre Mai--Emery s'est chargé de développer le fonctionnement globale de ce gestionnaire de film et donc d'implémenter les différentes fonctions de traitement, des fonctions necessaire à l'import/export d'un fichier .csv et de la fonction de listage ;
  • Mickaël Mai--Emery s'est chargé des différentes fonctions de traitement (informations, ajout, modification, suppression d'une donnée) ;

Toute la vérification et les tests pour vérifier si tout le code est fonctinonel et sans bugs a été réalisé par les deux membres du groupe.

Ce projet aura nécessité environ 20 heures de travail étalées sur 4 semaines. Que se soit sur la route d'un voyage ou dans notre bureau ou en cours, de nombreux moments ont été consacré à se projet pour le perfectionner, toujours en ajoutant de petites nouvelles idées et correctifs. La mise en commun du code a été facilité par l'utilisation de Firebase Studio, un outil créer par Gooogle. L'échange a été simplifié par le fait que nous habitons ensemble ce qui a aussi permis de faire des modification plus rapidement et plus facilement.

Fonctionnement

Fonctionnalités

Une fois que nous avons selectionner la base de données sur laquelle s'appuyer, le programme nous demande directement le choix du traitement à effectuer en nous listant les options possible. Pour sélectionner l'option choisi, il suffit de taper le numéro correpondant et comme dans tout le programme si le saisie est incorrecte le programme nous le spécifie.

Dans la partie "lister les films", nous avons le choix entre 7 types de tri ou classement (par nom, genre, etc...) ce qui permet de retrouver le film que l'on cherche plus facilement.

Ensuite dans la partie En savoir plus sur un film, nous avons aussi la possibilité de chercher un film avec un mot clé comme sur un moteur de recherche mais nous pouvons tout de même lister les films si nous ne connsaissons pas tous les films présent dans la base de données. Cela permet un accès aux informations plus rapides et surtout plus simple pour l'utilisateur.

Dans la partie "ajouter un film", le programme nous guide pour saisir toutes les informations nécessaire. Il nous suffit de répondre aux questions que le programme nous demande et le film est maintenant ajouté à la base de données.
C'est la même chose pour la fonction "modifier" et "supprimer" où le programme nous guide pour la modification du film en nous demandant quel est l'élément à modifier, etc...

Enfin, nous pouvons enregistrer nos modifications pour que les traitements que nous venons d'effectuer ne soient pas perdus !

Difficultés rencontrées

La séparation du code en plusieurs parties, l'organisation et la logique de l'outil a été difficile. Les nombreuses conditions et la gestion de la demande utilisateur ont rendu le code long et confu à certains moments

La partie la plus compliqué du code était que pour chaque demande à l'utilisateur, il fallait vérifier si la saisie était correct et qu'il ne fasse pas planter le gestionnaire de film. Pour la manipulation des données, nous demandions souvent un nombre entier avec int() mais si la saisie utilisateur était une chaine de caractère, le gestionnaire plantait et tout ce que nous avions fait était perdu. Nous avons donc trouver une solutions efficace afin de contourner l'erreur de Python lié au mauvais type de donnée : 


        def run():
            '''
            Fonction de lancement du programme

            reponse=0
            while not 1<=reponse<=6:
                # première partie de la correction de bug avec 'try:'
                try:
                    reponse = menu.demande()
                    # Conditions pour choisir le traitement
                    if reponse == 1:
                        menu.lister(films)
                    elif reponse == 2:
                        menu.infos(films)
                    elif reponse == 3:
                        menu.ajouter(films)
                    elif reponse == 4:
                        menu.modifier(films)
                    elif reponse == 5:
                        menu.supprimer(films)

                    # Redirection de la conditions si la saisie n'est pas bonne

                    print('\nLe choix doit être compris entre 1 et 6, réessayer')
                reponse=0
